快速开始¶

这份指南帮助你从一台新机器走到可运行的 Infernux 场景，当前针对 Windows 上的 0.2.1 预览版。预览版迭代很快；排查行为差异前，请先确认 Hub 或 Release 页面显示的实际版本。

你将完成什么¶

完成后，你应该能够：

  • 安装或启动 Infernux；
  • 创建或打开项目；
  • 理解 Scene、Game、Hierarchy、Inspector、Project 与 Console 面板的职责；
  • 保存场景并进入 Play 模式。

1. 获取引擎¶

请选择一种受支持的入口：

  1. 从 Infernux Releases 获取最新预览版。
  2. 按该版本包内的说明安装或解压。
  3. 学习阶段建议把 Hub、编辑器和项目放在本地可写磁盘中。

如果你的目标是开发引擎本身，请使用仓库里的源码构建说明，不要把源码构建与打包预览版混在同一套目录中。

2. 创建或打开项目¶

启动 Hub 或编辑器并选择项目位置。一个健康的项目应该能正常打开，在 Project 面板中显示文件，并且 Console 不会持续刷出同一个异常。

不要把引擎生成文件塞进无关源码仓库。独立项目目录会让资源发现、导入和故障排查简单很多。

3. 理解编辑器布局¶

区域 作用 第一次检查什么
Scene 编辑世界并可视化选择对象 能否环绕或聚焦场景？
Game 预览当前相机输出 进入 Play 后是否更新？
Hierarchy 查看当前场景对象 能否按名字选中对象？
Inspector 编辑选中对象及组件 切换选择时是否同步更新？
Project 浏览项目资源和脚本 能否找到项目资源文件？
Console 查看日志、警告和错误 是否存在反复出现的异常？

不同版本的停靠布局可能变化。稳定的心智模型是“面板名称和职责”，而不是某张截图中的固定位置。

4. 保存并运行场景¶

  1. 打开或创建一个场景。
  2. 添加或选择一个可见对象。
  3. 进入 Play 模式前保存场景。
  4. 启动 Play，观察 Game 面板和 Console。
  5. 对场景做需要保留的结构修改前，先退出 Play。

预期结果¶

编辑器能够干净地进入和退出 Play，Game 视图会更新，Console 不会每帧重复累积同一错误。

常见问题¶

编辑器无法启动¶

  • 确认下载包与 Windows 架构匹配。
  • 使用 Release 包提供的启动入口，不要单独移动其中的二进制文件。
  • 检查安全软件是否隔离了运行时依赖。

项目能打开但资源缺失¶

  • 确认打开的是项目根目录，而不是其中的资源子目录。
  • 编辑器导入资源时，不要同时在外部批量移动文件。
  • 优先阅读 Console 的第一条错误；后续错误可能只是连锁反应。

Play 后 Game 视图为空¶

  • 确认场景存在且启用了 Camera。
  • 确认实际运行的就是刚才编辑的场景。
  • 检查 Console 是否有脚本或资源加载错误。
